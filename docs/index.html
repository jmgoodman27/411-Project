Project Proposal- Jake Goodman

Project Description:

The project I want to create is a podcasts web app where the user can browse, search, and listen to podcasts online. There are many of these out there (although many more for mobile compared to web), but this one will be different. My podcasts application will be focused around individual episodes and will offer functionality to save and share these episodes. For most podcasts, not all episodes are equal in quality and this will help people save the episodes they like best and share them with friends.

Project Design:

The technologies I will use are Node.js, Express, and PostgreSQL. For queries I will use Knex.js, for the view engine I will use Nunjucks, and for authentication I will use Passport. I will also be using Sass for my CSS and Gulp.js to handle my builds. Lastly and most importantly, I will be using the iTunes Search API to retrieve podcasts in JSON format. This will give me the podcast information along with the RSS feeds which I can then parse with a Node library like rss-parser.

The project’s use cases (in order of importance) will be to:

  • Search podcasts by name
  • View list of individual episodes with the ability to save episodes to account and share with friends (will share RSS for that episode)
  • Browse podcasts (by name, rating, or genre which are all included in iTunes Search API)
  • View list of saved episodes and add additional info (like personal rating)

There may be additional use cases as the project develops.

My PostgreSQL database will be used to store user information including the episodes attached to each user (many-to-many). Everything else (most likely) will be retrieved through iTunes Search API.

Project Schedule:

Checkpoint 1- I will set up my project with all the technologies listed above along with the architecture for model, view, controller. I will build out basic views for each page that I want, but not necessarily with any functionality just yet. The user login system (with create an account, login, and profile pages) will be complete along with working sessions.

Checkpoint 2- I will work heavily on the search page and make sure I can successfully get and use all the data I need from the iTunes Search API. I will set up routes for clicking on a podcast and will work on the podcast page for displaying individual episodes. I will also begin to add functionality that allows for saving podcasts to the user’s account from this page.

Checkpoint 3- I will work on the browse page that will bring in data from iTunes Search API with options to sort by name, rating, and genre. I will work on the profile page to bring in saved episodes into a list and work on functionality to allow the user to rate each episode (and also sort by rating). I will make sure that all functionality from other checkpoints are fleshed out and that all use cases are met.

Final- The final will be everything from checkpoint 3 along with a coat of paint for design. I will also be sure to thoroughly test the web application for any bugs and give myself enough time to work on deployment of the app. I will then do any debugging in production and make sure it is ready to submit.

Project Justification:

Novelty: My project is different than existing applications for multiple reasons. There are not many web applications for podcasts as the bulk are for mobile, which would give people a new medium for discovering podcasts. Most podcast apps are centered around the podcast and the player. My app will focus on the individual episodes and a sharing element for these episodes. Being able to save, rate, and share favorite episodes allows for an enhanced podcasting experience.

Complexity: This app is plenty complex with all of the technologies I will be using and this project will allow me to gain more experience interacting with an API. I have been really wanting to dive deeper into Node.js and this is the perfect opportunity to do so. I have limited the scope of functionality to be feasible by the end of the semester, but think I could definitely build on this in the future if I wanted (for example, building a view for seeing top rated episodes from users).
