Project Description

The project I want to create is a podcasts web app where the user can browse, search, and listen to podcasts online. There are many of these out there, but this one will be based around a social element where the user can save, rate, share, and comment on episodes. For most podcasts, not all episodes are equal in quality and this will help people find the best episodes.

Project Design

The technologies I will use are Node.js, Express, and PostgreSQL. For queries I will use Knex.js, for the view engine I will use Nunjucks, and for authentication I will use Passport. I will also be using Sass for my CSS and Gulp.js to handle my builds. Lastly and most importantly, I will be using the iTunes Search API to retrieve podcasts in JSON format. This will give me the podcast information along with the RSS feeds which I can then parse with a Node library like rss-parser.

The projectâ€™s use cases will be to:

  • Search and browse podcasts by name, rating, or genre
  • View list of individual episodes with the ability to save, rate, share, and comment on episodes
  • View list of saved episodes, posted comments, and personal ratings on episodes

There may be additional use cases as the project develops.

My PostgreSQL database will have the following tables: user, comment, episode. Everything else (most likely) will be retrieved through the iTunes Search API.

Project Schedule

Checkpoint 1: Technologies listed above are set up along with an established MVC architecture. The user login system is complete with sessions and views including create an account, login, and profile.

Checkpoint 2: The search view is complete and retrieves podcast data from the iTunes Search API. The podcast view for displaying all episodes within a podcast is complete with the ability to play and save each episode. The friends page is complete with ability to search by username (and save a friend to a list) to see favorited episodes.

Checkpoint 3: The browse page is complete which brings in data from the iTunes Search API with options to sort by name, rating, and genre. The favorites page is complete which displays saved episodes with the ability to play, comment, and rate each episode directly from this page. When looking up a user from the friends page, these comments and ratings can also be seen. Any revisions and additional functionality will be complete by this checkpoint.

Final: The final will be everything from checkpoint 3 along with a coat of paint for design. At this point, the web application has been successfully deployed to the server and thoroughly tested for bugs.

Project Justification

Novelty: My project is different than existing podcast applications because it focuses on a social element that is missing from other podcast apps. There are also not many web applications for podcasts as the bulk are for mobile, which would give people a new medium for discovering podcasts.

Complexity: This app is plenty complex with all of the technologies I will be using and this project will allow me to gain more experience interacting with an API. I have been really wanting to dive deeper into Node.js and this is the perfect opportunity to do so.
